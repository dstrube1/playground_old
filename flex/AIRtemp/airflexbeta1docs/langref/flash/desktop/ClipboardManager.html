Packageflash.desktop
Classpublic class ClipboardManager
InheritanceClipboardManager Object

Provides access for copying data to and from the system clipboard.

All the members of the ClipboardManager are static, so an instance of the ClipBoardManager class does not need to be created

The data on the clipboard is contained in a TransferableData object. Different representations of the same information can be made available in a single TransferableData object to increase the likelihood that another application can understand and use that data. For example, an image might be included as image data, a serialized Bitmap object, and as a file. Rendering of the data in a format can be deferred so that the format is not actually created until the data in that format is read.

An AIR application object can be transfered as a reference and as a serialized object. References are only valid within the originating application. Serialized object transfers are valid between AIR applications, but can only be used with objects that remain valid when serialized and deserialized.

The clipboard contains a single data object. Whenever the clipboard is written to (inside or outside of an AIR application), the previous contents are no longer accessible.

Property Detail
dataproperty
data:TransferableData  [read-write]

A TransferableData object containing the contents of the clipboard.

You can only get or set the data property within a method passed as the closure parameter of the accessClipboard() method. If you attempt to set or get the data property elsewhere, the clipboard contents are not set or returned. The TransferableData object retrieved by getting the data value is only valid in the scope of the closure method. Saving a reference to it and calling its methods elsewhere, will generate runtime errors.

Once you have set the contents of the clipboard, the clipboard owns the TransferableData object and anything it contains. Do not mutate the object or its contents after calling set. If you do so, it may or may not change the state of the clipboard.

To clear the clipboard assign an empty TransferableData object to the data property. Note, that you do not need to clear the clipboard before putting new data on it.

Method Detail
accessClipboard()method
public static function accessClipboard(closure:Function):void

Provides access to the system clipboard.

Asynchronously acquires a system lock on the Clipboard, calls the function specified as the closure parameter, and then releases the lock. You must call this method to copy data to or from the clipboard.

Parameters

closure:Function — A function that reads or writes clipboard data using the ClipboardManager.data property.
