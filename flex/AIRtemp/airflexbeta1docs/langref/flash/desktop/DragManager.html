Classpublic class DragManager
InheritanceDragManager Object

Coordinates drag-and-drop operations. With the native drag-and-drop API, you can allow a user to drag data between an AIR application and the native operating system, between two AIR applications, as well as between components within a single application.

Supported transfer formats include:

  • Bitmaps
  • Files
  • Text
  • URL strings
  • Serialized objects
  • Object references (only valid within the originating application)

Note: all DragManager members are static. An instance of this class does not need to be created.

A drag-and-drop operation is a user interface gesture that begins with the user clicking on a visible item and dragging it elsewhere. During the drag gesture, objects on the display list dispatch NativeDragEvents as the gesture moves across the AIR application window. Handlers for these events can call the methods of the DragManager to indicate when a dragged item can be dropped. In response, the DragManager changes the mouse cursor to provide feedback to the user.

Drag Actions

Drag-and-drop gestures are typically used for three types of operations called actions. Since the meaning of these actions depends on the application context, the runtime does not enforce any particular behavior with respect to actions. However, properly implementing the actions will improve the user experience with your application.

The possible actions are:

  • copy: A copy of the data should be made, leaving the original untouched. When dragging objects within an application, care should be taken to copy the original object itself rather than the reference to that object.
  • move: The data should be moved from its original context into the context defined by the drop target, such as when moving an item from one list to another.
  • link: A shortcut or reference to the original data should be created, while still leaving the item within its original context.

The allowed actions can be set for a drag gesture by supplying an allowedActions parameter in the DragManager.doDrag() call that starts the drag operation. If no allowedActions parameter is provided, all of the actions are allowed. Potential drag targets can check which actions are allowed with the allowedActions property of a NativeDragEvent object, and should not accept a drop that only allows incompatible actions (this is not enforced by the runtime, however).

When accepting a drop, a potential drop target should indicate the action chosen by setting the DragManager.dropAction property in response to the nativeDragDrop event. This action is reported back to the initiating display object in the nativeDragComplete event. If no action is set by a drop target, then a default action, chosen from the allowed actions in this order of precedence: copy, move, link, is reported in the event. The initiating object is responsible for updating its internal state in response to the chosen action.

String constants for the action names are defined in the DragActions class.

Sequence of events

A drag gesture is begun by calling the DragManager.doDrag() method within a mouseDown or mouseMoved event handler and proceeds through the following event sequence in response to user actions:

  • nativeDragStart event: When DragManager.doDrag() is called, the DisplayObject passed as a paramter to the method becomes the initiator object and will dispatch a nativeDragStart event.
  • nativeDragEnter, nativeDragOver events: When a drag gesture passes over a DisplayObject, that object will dispatch a nativeDragEnter event. While the drag gesture remains over the DisplayObject, it will continually dispatch nativeDragOver events. In response to either of these events, an object that serves as a potential drop target should check the properties of the event object to decide whether it can accept the drop. If the data format and allowed actions are appropriate, then the event handler for these events must call DragManager.acceptDrop(), passing in a reference to display object that will serve as the drag target (typically the object that dispatched the event). The user will then be able to drop the dragged item onto the target.
  • nativeDragExit event: When a drag gesture passes out of a DisplayObject, then the object dispatches a nativeDragExit event. If the object had been designated as the drag target by an earlier call to the DragManager.acceptDrop() method, that call is no longer valid and acceptDrop() must be called again if the gesture re-enters the DisplayObject.
  • nativeDragDrop event: The target display object dispatches a nativeDragDrop event when the user releases the mouse button over the object. The handler for this event can access the data in the transferable property of the event object and should set the DragManager.dropAction property to signal which action should be taken by the initiator object.
  • nativeDragComplete: When the user releases the mouse at the end of a drag gesture, the initiator object dispatches a nativeDragComplete event (whether or not the drop itself was consumated). The handler for this event can check the dropAction property of the event object to determine what, if any, modification should be made to its internal data state, such as removing a dragged-out item from a list. If dropAction==DragActions.NONE, then the dragged item was not dropped on an eligible target.

Gestures between application

When a drag gesture enters an AIR application window from a non-AIR application, there will be no initiator object to dispatch the nativeDragStart or nativeDragComplete events. The events dispatched during the gesture will otherwise follow the same pattern as that of a gesture starting and ending within the same AIR application

When a drag gesture leaves an AIR application window, there will be no target object to dispatch nativeDragEnter, nativeDragOver, or nativeDragDrop events. The initiator object still dispatches a nativeDragComplete event, which reports the drag action set by the native operating system (or "none," if the drop was not accepted).

When a drag gesture moves from one AIR application to another, the initiator and target display objects dispatch events within their separate applications as usual.

Transfering information

The data transfered during a drag-and-drop gesture is contained in a TransferableData object. This data object is added to the drag operation with the DragManager.doDrag() method that starts the drag gesture. Potential drop targets can access the TransferableData object through the transferable property of the NativeDragEvent object. Once a drag operation has started, the TransferableData object can only be accessed in the event handler of a NativeDragEvent. Any other attempt to access the object will generate a runtime error.

Security considerations

The security sandboxes of the initiator and potential target objects determine how the the data being dragged can be accessed. If both objects are in the same sandbox, then the data can be accessed from any NativeDragEvent object. However, if the initiator and target objects are in different sandboxes, the data can only be accessed in the target sandbox within the event handler for the nativeDragDrop event. Other native drag event handlers can still still access the TransferableData object referenced in the eventtransferable property to determine which data formats are available, but calling the transferable.dataForFormat() method will generate a security error.

acceptDragDrop()method
public static function acceptDragDrop(target:DisplayObject):void

Informs the DragManager that the specified target DisplayObject can accept a drop corresponding to the current drag event.

This method should only be called when there is a nativeDragDrop handler on the specified target object that can handle at least one of the data formats in the dragged item and at least one of the allowed actions.

This function can be called only within a nativeDragEnter or nativeDragOver event handler.

doDrag()method 
public static function doDrag(initiator:DisplayObject, transferable:TransferableData, dragImage:BitmapData = null, offset:Point = null, actionsAllowed:DragOptions = null):void

Starts a drag-and-drop operation.

To start a drag operation:

  1. Create a new TransferableData object.
  2. Add the data to be transfered in one or more formats.
  3. Optionally, create a BitmapData object to serve as a proxy image during the drag.
  4. Optionally, create a DragOptions object to restrict the actions allowed in this operation. (If the allowedActions parameter is left null, all actions are allowed.)
  5. Call DragManager.doDrag().

The initiator object will receive a nativeDragStart event after this method is called and a nativeDragComplete event when the user releases the mouse to end the drag gesture. The handler for the nativeDragComplete event can check the dropAction property of the event to determine whether the drag-and-drop operation was successfully completed. If dropAction==DragActions.NONE, then the dragged item was not dropped on an eligible target.

This method may be called only from within a mouseDown or mouseMoved event handler. (If called in response to a mouseMoved event, the mouse button must also be down.)

Parameters

initiator:DisplayObject — typically the object from which the drag gesture began. Receives the nativeDragStart and nativeDragComplete events.
 
transferable:TransferableData — a clipboard-like container for data being dragged.
 
dragImage:BitmapData (default = null) — an optional proxy image displayed under the mouse cursor during the drag gesture. If null, no proxy image is displayed.
 
offset:Point (default = null) — the offset between the mouse hotspot and the top left corner of the dragImage. Negative coordinates will move the image up and to the left in relation to the hotspot. If null, the top, left corner of the drag image will be positioned at the mouse hotspot.
 
actionsAllowed:DragOptions (default = null) — restricts the drag-and-drop actions allowed for this operation. If null, all actions are allowed.

Event Object Type: flash.events.NativeDragEvent
NativeDragEvent.type property = flash.events.NativeDragEvent.NATIVE_DRAG_COMPLETE

Dispatched by the drag initiator DisplayObject when the user releases the drag gesture.

The event's dropAction property indicates the action set by the drag target object; a value of "none" (DragActions.NONE) indicates that the drop was canceled or was not accepted.

The nativeDragComplete event handler is a convenient place to update the state of the initiating display object, for example, by removing an item from a list (on an drag action of "move"), or by changing the visual properties.

NativeDragEvent.NATIVE_DRAG_COMPLETE defines the value of the type property of a nativeDragComplete event object.

This event has the following properties:

PropertyValue
actionsAllowedThe DragOptions object specifying the actions relevant to this drag operation.
bubblesfalse
cancelablefalse; there is no default behavior to cancel.
dropActionThe action chosen by the drop target (or the default action if no action was set).
transferableThe TransferableData object containing the dragged data.
