Packageflash.events
Classpublic class NativeWindowErrorEvent
InheritanceNativeWindowErrorEvent ErrorEvent TextEvent Event Object

Dispatched when an error causes a window operation to fail.

Note: this class is not currently used. NativeWindowErrorEvents are never dispatched.

You can check for error events that do not have any listeners by using the Flash Player Debugger version. Flash Player displays the string defined by the text parameter of the NativeWindowErrorEvent constructor.

NativeWindowErrorEvent()Constructor
public function NativeWindowErrorEvent(type:String, bubbles:Boolean = false, cancelable:Boolean = false, text:String = "", id:int = 0)

Creates an Event object that contains specific information about ioError events. Event objects are passed as parameters to Event listeners.

Parameters
type:String — The type of the event. Event listeners can access this information through the inherited type property. There is only one type of input output error event: WindowErrorEvent.IO_ERROR.
 
bubbles:Boolean (default = false) — Determines whether the Event object participates in the bubbling stage of the event flow. Event listeners can access this information through the inherited bubbles property.
 
cancelable:Boolean (default = false) — Determines whether the Event object can be cancelled. Event listeners can access this information through the inherited cancelable property.
 
text:String (default = "") — Text to be displayed as an error message. Event listeners can access this information through the text property.
 
id:int (default = 0) — A reference number to associate with the specific error.

Constant Detail
INVALID_BOUNDS_ERRORConstant
public static const INVALID_BOUNDS_ERROR:String = "invalidBoundsError"

Flash Player dispatches the invalidBoundsError event when the user code attempts to size or move the window to invalid values. Invalid values are dependent on the current security privileges on the running system, as well as limits managed by the underlying operating system.

Setting the width or height of a Window to zero or negative values will always raise an invalidBoundsError. Setting the dimensions of the window very large or moving the window location offscreen may also raise this error, depending on security settings.

Setting the bounds of a closed window will raise a CLOSED_WINDOW_ERROR regardless of whether there are bounds errors as well.

This event has the following characteristics:

CharacteristicDescription
TargetThe object on which the error occurred.
Default BehaviorNone.
BubblesNo.
CancelableNo. There is no default behavior to cancel.
Objects that generate this eventWindow.
Example listener functionfunction boundsErrorListener(event:WindowErrorEvent) {    trace(event.type); // output: networkError}
Example addEventListener() callsomeEventDispatcher.addEventListener(WindowErrorEvent.INVALID_BOUNDS_ERROR, boundsErrorListener );
